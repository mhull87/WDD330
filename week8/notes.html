HTML5 & CSS3 for the Real World Ch 8 CSS3 Transforms and Transitions

  • Transforms

      • Translate:

      • transform: translate(45px, -45px);
      • moves alsong the x and/or y axis

      • Scaling:

      • transform: scale(1.5, .23);
      • along the horizontial and vertical dimensions
      • scaling does not change the other elements, it could overlap

      • Rotation

      • transform: rotate(10deg) translateX(40px) scale(1.5);
      • values can be degrees, grads, radians, or turns
      • they will be exicuted in the order listed

      • Skew

      • transform: skew(15deg, 4deg)
    • Change the origin with transform-origin: top center; or 10% 10%;
    • default is center

    • Transitions

    • change the transformation
    • declare the origional state and the final state.
    • transition-property: transform;
    • transition-property: transform, color;
    • use as many as you want with a comma seperated list or all
    • transition-duration: .2s;
    • 200ms is the optimum time for a transition: anything slower will amke the website seem slow
    • transition-timing-function: ease out;
    • lets you control overall animation like fast first then slow
    • set it specifically with cubic-bezier()
    • steps(5, start) would jump through the equidistant steps of 0%, 20%, 40%, 60%, and 80%
    • steps(5, end) would go 30%, 40%, 60%, 80% and 100%
    • transition-delay: 50ms;
    • negative delays make some cool effects
    • shorthand:
    • transition: transform .2s ease-out 50ms;
    • delay goes after duration otherwise any order is fine

  • Animations

  • Keyframes allow you to set multiple changing points
  • @keyframes moveRight {
    from {
    transform: translateX(-50%);
    }
    to {
    transform: translateX(50%);
    }
    }

    @keyframes appearDissapear {
    0%, 100% {
    opacity: 0;
    }
    20%, 80% {
    opacity: 1;
    }
    }

    @keyframes bgMove {
    100% {
    background-position: 120% 0;
    }
    }
  • animation-name: appearDissapear;
  • animation-duration: 300ms;
  • animation-timing-function: linear;
  • animation-iteration-count: infinite; or any number of times
  • animation-direction: alternate; will run from 0% to 100% and then 100% back to 0% instead of starting over from 0%
  • animation-delay: 50ms;
  • animation-fill-made: none, forwards, backwards, or both
  • animation-play-state: running or paused;
  • Shorthand:
  • animation: 300ms ease-in alternate 5s backwards appearDisappear;
