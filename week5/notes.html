    • system error

  • Exceptions

    • an error that produces a return value that can be used by the program to deal with thte error

  • Warnings

  • Strict Mode

    • limits the use of some deprecated features
    • poor style is viewed as an error
    • can use strict mode on a perfunction basis by placing it inside the function
    • 'use strict'; //put this on the first line of the js file or prefered way:
    • (function() {
      'use strict';
      // all code here
      }());
    • modules use strict mode as default
    • Linting tools help 'clean up' code style

  • Debugging in the Browser

    • use alert();
    • use the console (console.log();)
    • console.trace(); will produce a stack trace
    • breakpoints with the debugger; keyword

  • Error Objects

    • const error - new Error('Oops, something went wrong');
    • EvalError is not used anymore
    • RangeError is thrown when a number is outside of the allowable range
    • ReferenceError is thrown when a reference is made to an item that doesn't exist
    • SyntaxError is thrown when there's an error in the code's syntax
    • TypeError is thrown when there's an error in the type of value used (a string where a number is expected)
    • URIError is thrown when there's a problem encoding or decoding the URI
    • InternalError is a non-standard error that is thrown when an error occurs in the JavaScript engine

  • Throwing Exceptions

    • throw your own exceptions using a throw statement
    • throw new Error('Something has gone badly wrong!';
    • function squareRoot(numbr) {
      'use strict';
      if (number < 0) {
      throw new RangeError('You cannot find the square root of negative numbers')
      }
      return Math.sqrt(number);
      };

      squareRoot(121); //11

      squareRoot(-1); //RangeError: You can't fine the square root of negative numbers

  • Exception Handling

    • You can catch the error keeping it hidden from the user but still identify it
    • try, catch, finally
      • function imaginarySquareRoot(number) {
        'use strict';
        try {
        return String(squareRoot(number));
        } catch(error) {
        return squareRoot(-number)+'i';
        }
        }
      • the catch block will only be run if there is an error otherwise it is skipped
      • imaginarySquareRoot(-49) //no error message shown //'7i'
      • a finally block can be added to always be run after the try or catch, regardless of which runs
      • function imaginarySquareRoot(number) {
        'use strict';
        let answer;
        try {
        answer = String(squareRoot(number));
        } catch(error) {
        answer = squareRoot(-number)+'i';
        } finally {
        return `+ or - ${answer}`;
        }
        }
      • always test your code and functions
    • Jest is fun run jest -c {} with a .test.js file
